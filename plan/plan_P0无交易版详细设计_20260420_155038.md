# P0 无交易版详细设计文档落盘方案

## 摘要
基于 `银发知识付费平台_PRD_v1.1.docx` 与现有 [P0-详细设计-无交易版.md](/Users/wangyu/local/workspace/silver-wisdom/backend/docs/P0-详细设计-无交易版.md)，输出一份“P0 无交易版”完整详细设计，并新建落盘到仓库根目录 `./plan/`。  
文档目标是可直接用于后端评审与拆解开发任务，覆盖模块归属、核心流程、接口契约、数据模型、Redis 设计、一致性、异常处理、测试与上线验收。  
本次明确排除交易域实现，不设计支付、订单、退款、对账，只在边界章节说明后续扩展点。

## 实施内容
1. 新建目录 `plan/`，生成新文件：
   `plan/plan_P0无交易版详细设计_<yyyyMMdd_HHmmss>.md`
2. 文档内容采用以下固定结构：
   - 文档背景与设计目标
   - 范围定义
   - 总体架构与模块职责
   - 端到端核心流程
   - 接口详细设计
   - 数据库详细设计
   - Redis 设计与一致性策略
   - 鉴权、安全与可靠性设计
   - 测试验证方案
   - 风险与缓解
   - 待确认项
3. 范围定义按“无交易版 P0”锁定：
   - 纳入：热点管理、分类管理、内容管理、AI 问答、AI 生成内容、兴趣标签采集、学习中心已生成内容查询
   - 不纳入：付费内容购买、AI 生成收费、订单支付、审核流、推荐系统、学习打卡、数据看板、历史多轮会话持久化
4. 模块职责按现有工程落位：
   - `silver-gateway`：`/api/**` 路由、Sa-Token 校验、统一限流、跨域
   - `silver-user`：微信登录、兴趣问卷、标签快照缓存
   - `silver-content`：热点、分类、内容、AI 问答、AI 生成任务、多模型网关抽象
   - `silver-course`：学习中心已生成内容聚合、阅读记录
   - `silver-common`：统一返回、异常、错误码、通用常量
5. 核心流程必须逐条展开为“调用链 + 状态变化 + 失败处理”：
   - 管理端热点发布
   - 管理端分类维护
   - 管理端内容发布
   - 小程序 AI 问答
   - AI 生成任务异步执行
   - 新用户兴趣采集
   - 学习中心查询已生成内容
6. 接口设计输出到字段级，统一采用 `/api/**` 网关前缀，至少覆盖：
   - `POST /api/user/auth/wx-login`
   - `POST /api/user/profile/onboarding`
   - `GET /api/user/profile/tags`
   - `GET /api/content/hotspots/active`
   - `GET /api/content/categories`
   - `GET /api/content/items`
   - `GET /api/content/items/{id}`
   - `POST /api/content/ai/chat`
   - `POST /api/content/ai/generate`
   - `GET /api/content/ai/generate/{taskId}`
   - `GET /api/course/learn/generated`
7. 公共接口契约统一写清：
   - 返回结构：`code`、`message`、`data`
   - 鉴权方式：`Authorization: Bearer <token>`
   - 错误码分层：参数错误、未登录、无权限、内容不存在、AI 供应商不可用、任务执行失败、系统异常
   - DTO/VO 字段约束：必填、长度、枚举值、默认值、可空策略
8. 数据库设计按表级落地，并补齐约束、索引、状态定义、审计字段：
   - `user_account`
   - `user_interest_tag`
   - `hotspot`
   - `category`
   - `content_item`
   - `content_tag_rel`
   - `ai_generate_task`
   - `ai_generate_result`
   - `learn_record`
   每张表明确主键、唯一键、查询索引、状态字段含义、创建更新时间。
9. Redis 设计必须明确 key、TTL、删除策略：
   - `hotspot:active:list`
   - `category:list`
   - `content:list:{categoryId}:{sortBy}:{pageNum}:{pageSize}`
   - `content:detail:{contentId}`
   - `user:tags:{userId}`
   - 可补充 `ai:provider:circuit:{provider}` 与 `ai:provider:rpm:{provider}:{yyyyMMddHHmm}` 作为无交易版多模型网关运行态缓存
10. AI 多模型网关在无交易版中保留为 `silver-content` 内部能力，不拆独立模块；设计内容包括：
   - 供应商配置模型
   - 加权轮询策略
   - RPM 限流计数
   - 最多 3 次失败切换
   - 连续失败熔断与半开恢复
   - 调用日志字段
   - 对 `AI 问答` 与 `AI 生成` 的统一接入方式
11. 文档中的接口、流程、数据对象名称使用可落地命名，不用伪代码；类/对象命名默认对齐 Java 分层，例如：
   - `WxLoginRequest`、`OnboardingProfileRequest`
   - `AiChatRequest`、`AiGenerateTaskStatusResponse`
   - `HotspotService`、`AiGatewayService`、`LearnRecordService`

## 公共接口与类型变更
- 新增统一认证语义：
  - 管理端与小程序端均基于 `Sa-Token`
  - 文档中区分 `loginType=admin` 与 `loginType=miniapp`
- 新增 AI 网关核心类型定义：
  - `AiProviderConfig`
  - `AiRouteContext`
  - `AiInvokeResult`
  - `AiGenerateTaskStatus`
- 新增内容域核心枚举定义：
  - `ContentType`: `ARTICLE` / `VIDEO`
  - `PublishStatus`: `DRAFT` / `PUBLISHED`
  - `GenerateTaskStatus`: `PENDING` / `RUNNING` / `SUCCESS` / `FAILED`
  - `LearnRecordType`: `GENERATED` / `CONTENT_VIEW`
- 新增错误码段规划：
  - `USER_***`
  - `CONTENT_***`
  - `AI_***`
  - `COURSE_***`

## 测试与验收场景
- 文档一致性检查：
  - 每个 P0 需求点都能映射到至少一个流程、一个接口或一个数据对象
  - 每个跨服务调用都写明超时、重试、降级或失败回退
- 接口级验证用例：
  - 登录成功与未登录拦截
  - 热点生效/失效窗口校验
  - 分类列表排序与状态过滤
  - 内容列表 `LATEST` / `HOT` 排序
  - AI 问答正常返回、供应商切换、兜底失败
  - AI 生成任务从 `PENDING` 到 `SUCCESS/FAILED`
  - 学习中心可查询生成结果
- 数据与缓存验证：
  - 热点/分类/内容变更后缓存失效
  - 用户兴趣问卷提交后标签快照刷新
  - AI 供应商 rpm 计数与熔断 key 生效
- 验收结果标准：
  - 文档可直接用于研发拆分接口和建表
  - 无交易边界清晰，不与 PRD 全量 P0 混淆
  - 所有关键对象、状态、错误码、缓存策略都有明确结论

## 假设与默认决策
- 本次以“无交易版 P0”为准，不补付费链路设计。
- Markdown 仅新建到仓库根目录 `plan/`，不覆盖现有 `backend/docs/P0-详细设计-无交易版.md`。
- 文档默认面向后端详细设计评审，不扩展前端页面交互稿。
- AI 多模型能力虽然源自 PRD 全量 P0，但在无交易版中仍保留，因为它直接服务 AI 问答与 AI 生成。
- 对象存储、第三方微信接口、OpenAI 供应商具体密钥配置只描述契约与配置项，不写真实环境值。
