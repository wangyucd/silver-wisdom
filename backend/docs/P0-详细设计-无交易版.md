# 银发知识付费平台 P0 详细设计文档（无交易版）

## 1. 文档摘要
本设计仅覆盖 P0，且不包含付费/交易相关能力。  
目标是在 1 个迭代内交付“管理端内容供给 + 小程序内容消费 + AI 问答与生成 + 基础用户画像 + 学习沉淀”闭环。

默认技术约束：
- JDK 17
- Spring Boot + Nacos + Redis + Sa-Token
- 模块：`silver-common`、`silver-gateway`、`silver-user`、`silver-content`、`silver-course`
- AI 供应商：OpenAI（固定接入）

## 2. 架构与模块设计
### 2.1 系统架构
- 管理端（Web）：热点、分类、内容发布维护。
- 小程序端：AI 问答、分类浏览、内容阅读、已生成内容学习。
- 网关层：统一入口、鉴权、路由、限流、跨域。
- 业务服务层：用户域、内容域、学习域。
- 基础设施层：MySQL（主数据）、Redis（高频缓存）、OpenAI（生成能力）。

### 2.2 模块职责
- `silver-gateway`
  - 统一 API 入口 `/api/**`
  - Token 校验（Sa-Token）
  - 路由转发与基础限流
- `silver-user`
  - 微信登录态接入
  - 兴趣问卷落库
  - 用户兴趣标签查询
- `silver-content`
  - 热点管理与查询
  - 分类管理与查询
  - 预置内容管理与查询
  - AI 问答、AI 生成任务
- `silver-course`
  - “学习中心-已生成内容”聚合查询
  - 阅读/查看行为记录
- `silver-common`
  - 统一响应 `R<T>`
  - 错误码、异常、全局异常处理
  - 通用常量与工具

## 3. 业务流程设计（P0）
### 3.1 热点发布流程（管理端）
- 运营录入热点：标题、摘要、封面、权重、上下线时间。
- 服务端校验时间与必填字段。
- 写入 `hotspot`，更新 Redis 热点缓存。
- 小程序首页读取有效热点，按权重降序展示首条为主热点。

### 3.2 内容发布流程（管理端）
- 运营创建内容：图文或视频，绑定分类与标签。
- 提交后直接发布（P0 无审核流）。
- 写入 `content_item` 与 `content_tag_rel`。
- 失效相关缓存（分类列表、内容列表缓存）。

### 3.3 小程序 AI 问答流程
- 用户提交问题到 `POST /content/ai/chat`。
- `silver-content` 组装提示词并调用 OpenAI。
- 返回回答正文与引用片段（若命中平台内容）。

### 3.4 AI 生成内容流程（无交易）
- 用户提交生成请求 `POST /content/ai/generate`。
- 生成任务写入 `ai_generate_task`（状态 `PENDING`）。
- 异步执行器调用 OpenAI，落地结果到 `ai_generate_result`。
- 任务状态更新为 `SUCCESS` 或 `FAILED`。
- 成功结果同步写入学习记录，学习中心可复看。

### 3.5 新用户兴趣采集流程
- 首次登录后进入 5 步问卷。
- 用户选择兴趣分类/话题。
- 写入 `user_interest_tag`，并缓存用户标签快照。

## 4. 接口设计（公共契约）
统一返回：
- `code`：0 成功，非 0 失败
- `message`：提示信息
- `data`：业务数据

鉴权约定：
- 登录后请求头：`Authorization: Bearer <token>`
- 除登录接口外均需登录态

### 4.1 用户域接口
1. `POST /api/user/auth/wx-login`
- 入参：`{ code: string }`
- 出参：`{ token: string, userId: long, isNewUser: boolean }`

2. `POST /api/user/profile/onboarding`
- 入参：`{ tags: string[], source: "QUESTIONNAIRE" }`
- 出参：`{ saved: true }`

3. `GET /api/user/profile/tags`
- 出参：`{ tags: [{ tag: string, weight: number }] }`

### 4.2 内容域接口
1. `GET /api/content/hotspots/active`
- 出参：`[{ id, title, summary, coverUrl, weight, startTime, endTime }]`

2. `GET /api/content/categories`
- 出参：`[{ id, name, iconUrl, coverUrl, sort }]`

3. `GET /api/content/items?categoryId=&sortBy=&pageNum=&pageSize=`
- `sortBy`：`LATEST` | `HOT`
- 出参：`{ total, list: [{ id, title, summary, coverUrl, type, heatScore, publishTime }] }`

4. `GET /api/content/items/{id}`
- 出参：`{ id, title, summary, type, contentBody, videoUrl, tags, categoryId }`

5. `POST /api/content/ai/chat`
- 入参：`{ question: string, contextItemIds?: long[] }`
- 出参：`{ answer: string, references: [{ contentId, title, snippet }] }`

6. `POST /api/content/ai/generate`
- 入参：`{ prompt: string, scene: string }`
- 出参：`{ taskId: string, status: "PENDING" }`

7. `GET /api/content/ai/generate/{taskId}`
- 出参：`{ taskId, status, result?: { title, body, outline }, failReason?: string }`

### 4.3 学习域接口
1. `GET /api/course/learn/generated?pageNum=&pageSize=`
- 出参：`{ total, list: [{ taskId, title, summary, createdAt }] }`

## 5. 数据库设计（MySQL）
### 5.1 用户相关
- `user_account`
  - `id` bigint PK
  - `wx_openid` varchar(64) unique
  - `nickname` varchar(64)
  - `avatar_url` varchar(255)
  - `status` tinyint
  - `created_at` datetime
  - `updated_at` datetime

- `user_interest_tag`
  - `id` bigint PK
  - `user_id` bigint
  - `tag` varchar(64)
  - `weight` decimal(5,2)
  - `source` varchar(32)
  - `created_at` datetime
  - `updated_at` datetime
  - 索引：`idx_user_tag(user_id, tag)`

### 5.2 内容相关
- `hotspot`
  - `id` bigint PK
  - `title` varchar(128)
  - `summary` varchar(512)
  - `cover_url` varchar(255)
  - `weight` int
  - `start_time` datetime
  - `end_time` datetime
  - `status` tinyint
  - `created_by` bigint
  - `created_at` datetime
  - `updated_at` datetime

- `category`
  - `id` bigint PK
  - `name` varchar(64) unique
  - `icon_url` varchar(255)
  - `cover_url` varchar(255)
  - `sort` int
  - `status` tinyint
  - `created_at` datetime
  - `updated_at` datetime

- `content_item`
  - `id` bigint PK
  - `title` varchar(128)
  - `summary` varchar(1000)
  - `type` varchar(16)（ARTICLE/VIDEO）
  - `category_id` bigint
  - `cover_url` varchar(255)
  - `content_body` longtext
  - `video_url` varchar(255)
  - `heat_score` int
  - `publish_status` tinyint
  - `publish_time` datetime
  - `created_by` bigint
  - `created_at` datetime
  - `updated_at` datetime
  - 索引：`idx_category_publish(category_id, publish_status, publish_time)`

- `content_tag_rel`
  - `id` bigint PK
  - `content_id` bigint
  - `tag` varchar(64)
  - 索引：`idx_content_tag(content_id, tag)`

### 5.3 AI 与学习相关
- `ai_generate_task`
  - `id` bigint PK
  - `task_id` varchar(64) unique
  - `user_id` bigint
  - `prompt` text
  - `scene` varchar(64)
  - `status` varchar(16)（PENDING/SUCCESS/FAILED）
  - `fail_reason` varchar(512)
  - `created_at` datetime
  - `updated_at` datetime

- `ai_generate_result`
  - `id` bigint PK
  - `task_id` varchar(64)
  - `title` varchar(128)
  - `summary` varchar(512)
  - `body` longtext
  - `created_at` datetime
  - 索引：`idx_task_id(task_id)`

- `learn_record`
  - `id` bigint PK
  - `user_id` bigint
  - `record_type` varchar(32)（GENERATED/CONTENT_VIEW）
  - `biz_id` varchar(64)
  - `progress` int
  - `last_view_time` datetime
  - `created_at` datetime
  - `updated_at` datetime
  - 索引：`idx_user_type_time(user_id, record_type, last_view_time)`

## 6. 缓存与一致性设计（Redis）
- Key 设计
  - `hotspot:active:list`
  - `category:list`
  - `content:list:{categoryId}:{sortBy}:{pageNum}:{pageSize}`
  - `content:detail:{contentId}`
  - `user:tags:{userId}`
- TTL 建议
  - 热点/分类：300s
  - 内容列表：180s
  - 内容详情：300s
  - 用户标签：600s
- 一致性策略
  - 内容/分类/热点写操作后主动删除相关缓存
  - 读取走 Cache-Aside 模式

## 7. 安全与非功能设计（P0）
- 鉴权：Sa-Token，区分管理端与小程序端登录类型。
- 数据安全：用户隐私字段脱敏日志输出，敏感配置放 Nacos 加密配置。
- AI 安全：生成文本做敏感词过滤与长度限制。
- 性能目标
  - 业务接口 TP99 < 500ms（不含 AI 生成耗时）
  - AI 问答首响应 < 3s（网关到首包）
- 可用性
  - AI 调用失败可重试 1 次，失败落库可追踪
  - 热点/分类接口优先读缓存，降低数据库抖动风险

## 8. 测试与验收用例（P0）
- 管理端发布热点后，小程序首页 1 分钟内显示最新有效热点。
- 管理端新增分类后，小程序分类页可见并可进入内容列表。
- 发布图文与视频内容后，详情页分别可正确展示正文与视频地址。
- 新用户完成兴趣问卷后，`GET /user/profile/tags` 返回正确标签。
- AI 问答输入常见问题，返回结构含 `answer` 与 `references` 字段。
- AI 生成任务从 `PENDING` 到 `SUCCESS`，学习中心可查询到该条记录。
- 未登录访问受保护资源返回鉴权失败；登录后访问成功。

## 9. 明确不包含（本期边界）
- 付费、订单、支付、退款、对账、交易流水
- 内容审核流（通过/驳回）
- 动态画像与协同过滤推荐
- 学习打卡、成就体系、运营数据看板
- AI 历史多轮会话持久化

## 10. 实施默认项（避免实现歧义）
- 内容发布状态：P0 仅 `DRAFT` 与 `PUBLISHED` 两态。
- AI 生成执行：异步任务线程池，失败可重试一次。
- 排序规则：`LATEST` 按 `publish_time desc`，`HOT` 按 `heat_score desc, publish_time desc`。
- 小程序端默认无搜索能力，先保证分类浏览闭环。
