# 银发知识付费平台 P0 详细设计文档（无交易版）

## 1. 文档摘要
本设计覆盖 P0 无交易版能力，目标是在 1 个迭代内交付“管理端内容供给 + 双端登录鉴权 + 用户管理 + 小程序内容消费 + AI 问答与生成 + 基础用户画像 + 学习沉淀”闭环。

默认技术约束：
- JDK 17
- Spring Boot + Nacos + Redis + Sa-Token
- 模块：`silver-common`、`silver-gateway`、`silver-user`、`silver-content`、`silver-course`
- AI 供应商：P0 固定接入 OpenAI，但保留多供应商扩展模型

本版新增明确角色：
- `pc-admin`：登录管理端的管理员，负责热点、分类、内容、用户管理
- `miniapp-user`：登录微信小程序的普通用户，负责内容消费、AI 问答、AI 生成、学习沉淀

## 2. 范围定义
### 2.1 P0 包含
- 管理端管理员账号登录
- 小程序微信登录
- 管理端用户管理
- 热点管理
- 分类管理
- 内容管理
- 小程序 AI 问答
- AI 生成内容
- 新用户兴趣标签采集
- 学习中心已生成内容查询

### 2.2 P0 不包含
- 订单、支付、退款、对账、交易流水
- 内容审核流
- 动态画像与协同过滤推荐
- 学习打卡、成就体系、运营数据看板
- AI 历史多轮会话持久化
- 管理员 RBAC 细粒度权限模型

### 2.3 本次关键假设
- 管理端管理员采用“账号 + 密码”登录，后续如需短信或企业 SSO 再扩展。
- 小程序用户采用微信小程序 `code2Session` 换取 `openid` 登录。
- P0 用户管理聚焦“查询、详情、启停用、查看标签与学习概况”，不做复杂客户运营体系。

## 4. 核心流程设计
### 4.1 管理员登录流程
1. 管理员在 `pc-admin` 输入账号与密码，调用 `POST /api/user/admin/auth/login`。
2. `silver-user` 校验账号状态、密码哈希、登录端类型。
3. 校验通过后通过 `Sa-Token` 生成 `admin` 登录态，写入管理员会话信息。
4. 返回 `token`、管理员基础信息、菜单权限占位字段。
5. 后续管理端请求统一携带 `Authorization: Bearer <token>`。

失败处理：
- 账号不存在或密码错误，返回 `USER_ADMIN_LOGIN_FAIL`
- 管理员账号被禁用，返回 `USER_ADMIN_DISABLED`
- 连续密码错误达到阈值，触发短期限流锁定

### 4.2 小程序用户登录流程
1. 用户在微信小程序触发登录，前端获取 `code`，调用 `POST /api/user/auth/wx-login`。
2. `silver-user` 调用微信 `code2Session` 获取 `openid`。
3. 若 `openid` 首次出现，则创建 `user_account` 并标记 `isNewUser=true`。
4. 生成 `miniapp` 登录态，返回 `token`、`userId`、`isNewUser`。
5. 新用户首次登录后跳转兴趣问卷。

失败处理：
- `code` 为空或失效，返回 `USER_WX_CODE_INVALID`
- 微信接口异常，返回 `USER_WX_LOGIN_FAIL`
- 用户被禁用，返回 `USER_ACCOUNT_DISABLED`

### 4.3 管理端用户管理流程
1. 管理员进入“用户管理”列表页，调用 `GET /api/user/admin/users` 查询分页结果。
2. `silver-user` 按昵称、手机号、状态、注册时间区间查询 `user_account`。
3. 列表返回用户基础信息、标签摘要、最近登录时间、状态。
4. 管理员点击详情页，调用 `GET /api/user/admin/users/{userId}`。
5. 详情页展示基础资料、兴趣标签、学习概况、最近生成记录摘要。
6. 管理员可执行启用/禁用操作，调用 `POST /api/user/admin/users/{userId}/status`。

失败处理：
- 非管理员访问返回 `USER_ADMIN_FORBIDDEN`
- 用户不存在返回 `USER_ACCOUNT_NOT_FOUND`
- 禁用后用户下一次访问受保护资源时被拒绝

### 4.4 热点发布流程（管理端）
1. 运营录入热点：标题、摘要、封面、权重、上下线时间。
2. 服务端校验时间与必填字段。
3. 写入 `hotspot`，更新 Redis 热点缓存。
4. 小程序首页读取有效热点，按权重降序展示首条为主热点。

### 4.5 内容发布流程（管理端）
1. 运营创建内容：图文或视频，绑定分类与标签。
2. 提交后直接发布，P0 不经过审核流。
3. 写入 `content_item` 与 `content_tag_rel`。
4. 删除分类列表、内容列表、详情缓存。

### 4.6 小程序 AI 问答流程
1. 用户提交问题到 `POST /api/content/ai/chat`。
2. `silver-content` 基于用户标签、上下文内容、平台预置内容构建提示词。
3. AI 网关选择可用供应商并发起调用。
4. 返回回答正文与引用片段。

### 4.7 AI 生成内容流程（无交易）
1. 用户提交生成请求 `POST /api/content/ai/generate`。
2. 生成任务写入 `ai_generate_task`，状态初始化为 `PENDING`。
3. 异步执行器取任务并调用 AI 网关。
4. 结果落地 `ai_generate_result`，任务更新为 `SUCCESS` 或 `FAILED`。
5. 成功结果同步写入 `learn_record`，供学习中心复看。

### 4.8 新用户兴趣采集流程
1. 新用户首次登录后进入 5 步以内问卷。
2. 用户选择兴趣分类/话题并提交。
3. `silver-user` 写入 `user_interest_tag`。
4. 刷新 Redis 用户标签快照 `user:tags:{userId}`。

## 5. 接口设计
### 5.1 公共契约
统一返回：
- `code`：200 成功，非 200 失败
- `message`：提示信息
- `data`：业务数据

统一鉴权：
- 请求头：`Authorization: Bearer <token>`
- 管理端接口仅允许 `loginType=admin`
- 小程序接口仅允许 `loginType=miniapp`

`POST /api/user/admin/auth/login`

## 7. Redis 设计与一致性策略
### 7.1 Key 设计
- `hotspot:active:list`
- `category:list`
- `content:list:{categoryId}:{sortBy}:{pageNum}:{pageSize}`
- `content:detail:{contentId}`
- `user:tags:{userId}`
- `user:status:{userId}`
- `user:session:latest-login:{userId}`
- `admin:login:fail:{username}`
- `ai:provider:circuit:{provider}`
- `ai:provider:rpm:{provider}:{yyyyMMddHHmm}`

### 7.2 TTL 建议
- 热点/分类：300s
- 内容列表：180s
- 内容详情：300s
- 用户标签：600s
- 用户状态缓存：300s
- 管理员登录失败计数：900s
- AI rpm 计数：70s
- AI 熔断标记：60s

### 7.3 一致性策略
- 内容/分类/热点写操作后主动删除相关缓存，读取采用 Cache-Aside。
- 用户标签更新后删除 `user:tags:{userId}`，由下次读取回填。
- 用户被管理员禁用后，同时刷新 `user:status:{userId}` 缓存。
- 业务接口在鉴权通过后增加一次用户状态校验，防止旧 token 越权访问。

## 8. 认证、安全与可靠性设计
### 8.1 Sa-Token 设计
- 管理端：`StpLogic("admin")`
- 小程序端：`StpLogic("miniapp")`
- 网关不直接做复杂权限判断，只做 token 透传与公共拦截。
- 各服务在 Controller 层或拦截器层校验登录类型与账号状态。

### 8.2 登录安全
- 管理员密码存储采用 `BCrypt` 哈希。
- 管理员登录失败连续 5 次，锁定 15 分钟。
- 微信登录接口限制单 IP 短时频率。
- 所有日志禁止输出明文密码、完整 openid、完整 token。

### 8.3 AI 调用可靠性
- 单次 AI 请求最多切换 3 次供应商。
- 连续失败 5 次触发熔断，熔断 60 秒。
- 所有 AI 失败场景落库，供后续追踪。

### 8.4 非功能指标
- 业务接口 TP99 < 500ms
- AI 问答首响应 < 3s
- 核心服务 SLA >= 99.9%

## 9. 测试与验收用例
### 9.1 登录与鉴权
- 管理员账号密码正确可登录成功，并拿到 `admin` 类型 token。
- 管理员密码错误返回失败，连续错误达到阈值后被短时锁定。
- 小程序用户 `code` 登录成功，首次登录返回 `isNewUser=true`。
- 管理员 token 访问小程序接口被拒绝，小程序 token 访问管理端接口被拒绝。

### 9.2 用户管理
- 管理员可分页查询用户列表，并按状态和注册时间过滤。
- 管理员可查看用户详情，详情包含标签与学习概况。
- 管理员禁用用户后，该用户再次访问受保护接口返回禁用提示。
- 管理员重新启用用户后，用户可恢复访问。

### 9.3 内容与学习
- 管理端发布热点后，小程序首页 1 分钟内显示最新有效热点。
- 管理端新增分类后，小程序分类页可见并可进入内容列表。
- 发布图文与视频内容后，详情页分别可正确展示正文与视频地址。
- 新用户完成兴趣问卷后，`GET /api/user/profile/tags` 返回正确标签。
- AI 问答输入常见问题，返回结构含 `answer` 与 `references` 字段。
- AI 生成任务从 `PENDING` 到 `SUCCESS`，学习中心可查询到该条记录。

## 10. 风险与缓解
### 10.1 微信登录依赖外部平台
- 风险：`code2Session` 不稳定会影响小程序登录。
- 缓解：保留 mock 配置与异常降级提示；增加接口告警。

### 10.2 管理员与用户双登录态混用
- 风险：错误复用 token 导致越权。
- 缓解：明确区分 `loginType`，接口按端隔离，统一拦截校验。

### 10.3 用户被禁用后旧 token 仍在有效期内
- 风险：禁用后短时间仍可访问。
- 缓解：业务接口增加用户状态校验，并通过 Redis 缓存快速拦截。

### 10.4 用户管理统计依赖跨服务聚合
- 风险：用户详情页学习概况需要依赖 `silver-course` 聚合。
- 缓解：P0 先采用同步查询，超时则降级返回基础资料和标签，学习概况置空。
