# silver-wisdom 项目级提示词

## 1. 项目基础约束
- JDK 版本必须使用 `17`。
- 后端技术栈默认使用：`Spring Boot`、`Nacos`、`Redis`、`Sa-Token`。
- 未经明确要求，不引入与上述栈冲突的替代方案。

## 2. 模块划分
项目采用以下模块（按你提供的清单创建）：
- `silver-common`：公共工具、通用配置、统一响应、基础常量与异常定义。
- `silver-gateway`：网关层，负责路由转发、统一鉴权入口、限流与跨域处理。
- `silver-user`：用户域服务，负责用户账号、登录态、权限相关业务。
- `silver-content`：内容域服务，负责内容发布、内容管理与查询能力。
- `silver-course`：课程域服务，负责课程信息、章节结构、课程状态管理。

## 3. 开发输出默认规范
- 所有新代码默认兼容 `JDK 17` 语法与特性。
- 优先输出清晰的分层结构（`controller`/`service`/`mapper` 或等价分层）。
- 涉及认证鉴权时，默认采用 `Sa-Token` 方案设计。
- 涉及缓存时，默认采用 `Redis`，并说明 key 设计与过期策略。
- 涉及服务注册发现与配置管理时，默认采用 `Nacos`。

## 4. 沟通与交付要求
- 方案说明需包含：模块归属、核心流程、关键接口、数据对象。
- 代码示例需给出可落地的类名、方法名、字段名，避免只给伪代码。
- 如需求与本提示词冲突，以用户当次明确指令为最高优先级。
